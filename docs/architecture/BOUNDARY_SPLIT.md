# Boundary Split

## Purpose

Define what this repository owns and what it explicitly does not own.

## Owned By This Repo

```yaml
owned_by_this_repo:
  - governance docs
  - task packets
  - validation scripts
  - project-specific source code
  - tests and fixtures
  - schemas and contracts
```

## Externalized To Other App Or Manual Workflow

```yaml
externalized:
  - raw source collection
  - messy file preparation
  - production deployment
  - credential provisioning
  - billing decisions
```

## Future-only

Future-only behavior must be labeled as future-only and must not be implied as current behavior.

## Closed Gates

Closed gates require explicit issue or human instruction before implementation.
